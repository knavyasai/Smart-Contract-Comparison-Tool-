## üéØ Project Goal & Impact

The primary objective was to accelerate the tedious process of comparing standard clauses across multiple legal documents (PDF/DOCX).

* **Domain:** Legal Technology / Smart Contract Review.
* **Quantified Impact:** The tool reduces manual review time for identifying corresponding clauses by an estimated **80%**.
* **Function:** Automatically ingests documents, finds similar paragraphs, and generates an auditable report detailing the differences.

---

## ‚öôÔ∏è Methodology & Technology Stack

The tool uses a two-stage Natural Language Processing (NLP) methodology:

### 1. Similarity Detection (Quantification)
* **Method:** **TF-IDF (Term Frequency-Inverse Document Frequency)** is used to transform text into numerical vectors, representing the importance of each word.
* **Algorithm:** **Cosine Similarity** is applied to the document vectors to calculate a numerical score (0.0 to 1.0) for every paragraph pair, identifying matching clauses. 

[Image of Cosine Similarity calculation in text vectors]


### 2. Difference Summarization (Qualitative Analysis)
* **Library:** **spaCy** is used for advanced text analysis on highly similar clauses.
* **Function:** It extracts and compares key **Named Entities** (e.g., dates, parties, locations) and important **Keywords** to create a concise summary that highlights the **specific points of divergence** between the two paragraphs.

### **Tech Stack**
* **Core:** Python
* **NLP:** `scikit-learn` (TF-IDF, Cosine Similarity), `spaCy`, `nltk`
* **File Handling:** `PyMuPDF` (PDFs), `python-docx` (DOCX), `os`
* **Data:** `pandas`
